# A consent guard on promotion to global scope

Refines ADR 0013. That decision made scope client-safe by default — project is the default, global is earned by client-neutrality — but whether a fact is safe to travel is entirely the model's judgment, and for material under an NDA the model's judgment alone is not enough. This adds a two-tier guard at the moment a Concept would be promoted to global scope. Content the judgment rules classify as **sensitive** — material carrying an explicit confidentiality signal: a stated confidentiality, an NDA or a non-disclosure term — is never promoted to global without the user's **explicit consent**. The classifier is deliberately **tight** (ask-rarely, set-and-forget): the axis is "is this confidential?", not "is this about a client?", so a plain client identity — a name, an email, a phone number — is not, on its own, sensitive, and is promoted like any other client-neutral fact unless it was explicitly flagged confidential. Because distillation runs unattended at session boundaries, the consent question is posed at the next session start and the content waits, project-bound, until answered; the safe state is therefore also the waiting state, so an unanswered consent never promotes the fact to global — the held Concept stays project-scoped and never travels as permanent memory. When the user triggers distillation manually ("distill now", ADR 0023) they are present, so the consent is resolved in the moment. Everything else is promoted automatically as before — 0% effort preserved — but every new global Concept is **announced** at the next session start and can be demoted to project scope or forgotten with one command. The sensitivity rules live in the editable judgment rules (ADR 0018), and the guard is deliberately minimal: one question, one answer, no policy engine.

## Considered Options

- **Automatic global promotion on the model's classification alone** (ADR 0013) — rejected as insufficient: an NDA needs more than the model's judgment before client facts travel.
- **Confirm every global promotion** — rejected: it destroys the 0%-effort promise for the common, safe case.
- **Sensitive requires consent (default is waiting); the rest is announced and reversible** (chosen).

## Consequences

- A verifiable gate on the **promotion channel**: a sensitive fact awaiting consent is not promoted to global until consent is given, so the held Concept — project-scoped like any other — never appears as a global Concept in another project's widened recall (Stage 5b). The gate sits on distillation's promote-to-global decision (`mimer.leakage`, `mimer.distill`), not on the raw record.
- **Scope of the gate — the raw long-term log is a separate, widenable channel (ADR 0013).** The confidential utterance the fact was distilled from is already captured, verbatim, into its origin project's daily log at capture time — before distillation runs at all — and that log is reachable by another project's `--widen`ed recall by ADR 0013's design. Redaction strips shape-detectable secrets, not confidential prose, so the guard neither does nor — being a promotion-time Concept gate — can suppress the raw-log chunk. The "never travels" guarantee is therefore about the promotion channel: a held fact never becomes a *global Concept* without consent. It is not a claim that the captured wording is unreachable across projects; a client who needs that must exclude the origin project from widening (ADR 0013) or turn its capture off. Closing the raw-log channel for held facts — suppressing a held fact's origin log chunks from widened recall until consent — is a possible future extension, out of scope for this minimal guard.
- The announcement of new global Concepts, with a one-command reversal, is the audit-and-undo path; the read-only CLI browser (ADR 0028) is where the user inspects, with their own eyes, what has become global.
