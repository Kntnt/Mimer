# Framing at the presentation seam: every surface shows stored memory through one audience-aware operation

ADR 0014 made "memory is data, not instructions" a boundary with two primitives — `frame` wraps injected content in a nonce fence under the data-frame header, `neutralise` defangs an untrusted leaf — but it left the rule enforced per caller. Four surfaces hand-wire the composition in four different shapes: the snapshot frames the whole payload (`snapshot.build_snapshot`), recall frames a neutralised excerpt block (`recall.RecallResult.rendered`), the management listing does the same (`manage._print_concepts`), and the profile neutralises each body without framing (`bundle.render_profile`), leaving the snapshot to frame it. A fifth or sixth surface must re-derive the rule or silently re-present memory as instructions — the same smeared invariant the write seam (ADR 0020) and the Visible seam (issue #54) each converged. This makes it structural: one operation, **`present`**, is the single path every surface that shows stored memory routes through, and it is **audience-aware**. For an **agent** audience it neutralises each untrusted leaf and wraps the whole in the data frame; for a **human** audience — the read-only browser (ADR 0028) — it shows memory verbatim, because a human is not injection-vulnerable, the data frame is only screen noise, and neutralising would corrupt the user's own Markdown. The seam distinguishes **trusted chrome** — Mimer's own labels, headings, citation brackets and age annotations — from **untrusted leaves**, and the rule is: any stored-derived value is a leaf, including the source and heading interpolated into a citation tag. Leaves are neutralised, never the assembled whole: `neutralise` strips every line-leading heading, so applied to the whole payload it would strip Mimer's own `## Profile` / `### title` structure — the leaf-vs-whole split is load-bearing and cannot collapse into a single `frame(neutralise(text))`. Bypass is policed exactly as ADR 0020 polices the write seam: `frame` and `neutralise` may be used only inside the seam, enforced by a living source-scan guard (the sibling of `tests/test_source_scan.py`) that fails any use of the raw primitives elsewhere, with documented exemptions for the session-boundary pass's clean-at-rest neutralisation and the digest prompt's `fence_transcript`. The seam is **self-sufficient**: it re-neutralises at show-time and does not depend on ingest-side neutralisation for its guarantee.

## Considered Options

- **`frame`/`neutralise` remembered at each surface** (the prior state) — rejected: four hand-wired shapes, and a new surface silently re-derives or forgets the rule. The proof is a latent gap this decision closes: `build_snapshot` places the manifest headlines and the distilled-since titles inside the frame un-neutralised, and `frame` strips fence brackets, the header signature and system-reminder tags but not line-leading headings — so a Concept titled `# …` could ride into the agent's framed payload as a live heading.
- **A single `present_memory(text) = frame(neutralise(text))`** over the assembled payload — rejected: `neutralise` would strip Mimer's own structural headings along with the untrusted ones. The leaf/whole distinction is essential, not incidental.
- **A type barrier** — stored-memory reads return an `Untrusted` value only the seam can render to a displayable string, making bypass structurally impossible as `storeio` makes an unredacted write impossible — rejected as disproportionate for this issue's smaller stakes: it threads a wrapper type through every read path unrelated to presentation. The source-scan guard delivers the "not a skippable helper" bar without it. Recorded so the heavier option is not re-proposed as "the obviously safer design"; it remains available if the stakes rise.
- **One audience** — rejected: the human read surface is not injection-vulnerable, and framing or neutralising there is noise that corrupts the user's own text. The seam is audience-parameterised.
- **A presentation seam: `present(segments, audience)`, leaves neutralised by construction, the whole framed for the agent audience only, primitives confined by a living source-scan guard** (chosen).

## Consequences

- `frame` and `neutralise` become internal to the seam; every other surface calls `present`. The source-scan guard fails any use of the raw primitives outside the seam, with the two documented exemptions above — the same "structural guarantee plus documented exemption" shape as ADR 0020.
- Five surfaces route through it: the snapshot (agent), recall (agent), the management Concept listing (agent), the SessionStart sensitive-promotion consent question (agent, #68) and `mimer-browse` (human, #67). `render_profile`'s per-leaf neutralisation folds into the seam as the snapshot's profile leaves.
- Mimer's own narration sits outside the frame uniformly — recall and the management listing already keep their trusted voice outside; the snapshot's announcement is brought into line — so only stored memory is labelled "data, not instructions".
- Ingest-side neutralisation stays as defence-in-depth (clean-at-rest), never pruned — the same stance ADR 0020 takes on the kept sink redaction. Its home moves onto the session-boundary pass when #63 folds the digest in; the stays-or-goes decision is unchanged, only the home moves.
- This is a cross-cutting refactor over exactly the surfaces #63/#67/#68/#69/#70 rewrite, so it is built after that rework lands, not concurrently, to avoid a semantic rebase on shared modules.
